# Requirements: Gitanjali Flutter Layout Migration

> Version: 1.0
> Status: APPROVED
> Last Updated: 2026-05-01

## Problem Statement

Существуют два устаревших iOS приложения (Objective-C, не Swift) с одинаковым функционалом на русском и английском языках:
- `legacy-gitabjali-ru-swift` - русская версия Sri Gaudiya Gitanjali
- `legacy-gitanjali-en-swift` - английская версия Sri Gaudiya Gitanjali

Необходимо перенести всю верстку, ассеты (изображения, аудио, шрифты) и UI в единое Flutter приложение с адаптивной версткой для всех размеров экранов.

## User Stories

### Primary

**As a** читатель духовной литературы
**I want** читать Sri Gaudiya Gitanjali на своем устройстве
**So that** я могу изучать тексты в удобное время на любом устройстве

**As a** пользователь с разными устройствами
**I want** адаптивный интерфейс
**So that** приложение выглядит хорошо на телефоне, планшете и десктопе

**As a** двуязычный пользователь
**I want** переключаться между русским и английским
**So that** я могу читать на предпочитаемом языке

### Secondary

**As a** пользователь
**I want** слушать аудио записи песен
**So that** я могу изучать правильное произношение

**As a** пользователь
**I want** сохранять закладки
**So that** я могу вернуться к важным местам

**As a** пользователь
**I want** искать по тексту
**So that** я могу найти нужные места в книге

## Acceptance Criteria

### Must Have

1. **Given** приложение запущено на любом устройстве
   **When** пользователь открывает страницу книги
   **Then** контент отображается корректно с адаптацией под размер экрана

2. **Given** пользователь на ЛЮБОМ устройстве (включая старые, маленькие экраны)
   **When** просматривает страницу
   **Then** макет автоматически адаптируется под размер экрана

3. **Given** пользователь на маленьком экране (< 360px ширины)
   **When** просматривает страницу
   **Then** контент использует минимальные отступы, текст читаем

4. **Given** пользователь на большом экране (> 900px ширины)
   **When** просматривает страницу
   **Then** контент центрирован с максимальной шириной для комфортного чтения

5. **Given** система определяет русский язык устройства
   **When** приложение загружается
   **Then** отображается русский контент (example_ru.xml)

6. **Given** система определяет другой язык устройства
   **When** приложение загружается
   **Then** отображается английский контент (example.xml)

7. **Given** страница имеет связанное аудио
   **When** пользователь нажимает play
   **Then** аудио воспроизводится с отображением прогресса

8. **Given** пользователь на любой странице
   **When** свайпает влево/вправо
   **Then** переходит на следующую/предыдущую страницу

### Should Have

9. **Given** приложение в portrait/landscape режиме
   **When** ориентация меняется
   **Then** layout плавно адаптируется

10. **Given** пользователь ищет текст
    **When** вводит запрос
    **Then** результаты показывают совпадения в контенте, комментариях и заголовках

11. **Given** пользователь добавляет закладку
    **When** перезапускает приложение
    **Then** закладки сохранены

12. **Given** пользователь хочет сменить язык
    **When** открывает настройки и выбирает другой язык
    **Then** приложение перезагружает контент на выбранном языке

13. **Given** пользователь ранее выбрал язык вручную
    **When** перезапускает приложение
    **Then** используется сохраненный выбор, а не системная локаль

### Won't Have (This Iteration)

- Онлайн синхронизация данных
- Авторизация пользователей
- Покупка дополнительного контента
- Push-уведомления
- Поддержка других книг (cookbook, avadhuta) - только Gitanjali

## Constraints

- **Technical**: Использовать существующую архитектуру Flutter приложения в `app/gitanjali/`
- **Assets**: Все изображения и аудио уже перенесены в `assets/legacy/`
- **Platform**: iOS, Android, Web, macOS, Windows, Linux
- **Font**: MurariChandUni.ttf для Sanskrit/Devanagari текста
- **Dependencies**: Минимизировать новые зависимости
